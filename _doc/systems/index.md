# System Engineering
: 시스템 전체 생명주기의 다양한 구성요소와 이해관계자들을 통합적으로 관리하는 접근 방식  
: 시스템 생명 주기는 요구사항 도출, 설계, 구성, 구현, 통합, 검증, 배포, 유지보수, 폐기까지 전 과정을 포함함  
: 구성 요소란 하드웨어, 소프트웨어, 데이터, 프로세스, 사람, 환경, 정책 등 시스템을 구성하는 모든 요소를 의미함  

**System**  
: 하나의 목적을 달성하기 위한 하드웨어, 소프트웨어, 프로세스 등이 상호작용하는 기능적 단위  
: 여러 시스템이 모여 협력하여 더 큰 목적을 달성하는 시스템 구조를 System of Systems(SoS) 라고 함  

**Software Engineering vs System Engineering**  
: 소프트웨어 엔지니어링은 소프트웨어의 설계, 구현, 테스트 등 소프트웨어 제품에 초점  
: 시스템 엔지니어링은 하드웨어, 소프트웨어, 프로세스, 사람, 정책 등 전체 시스템에 초점  


- 시스템 생명주기
- 시스템 프로세스
- 시스템 아키텍처
- 시스템 유형
- 시스템 방법론
- 시스템 관리


**history**
```
1940년대  벨 연구소에서 전화 시스템을 설계 및 관리하기 위해 처음으로 시스템 엔지니어링이라는 용어 사용
1950년대  미 국방부와 NASA에서 체계적인 프로젝트 관리를 위해 시스템 엔지니어링 방법론을 공식화
1970년대  컴퓨터가 발전하며 시스템 엔지니어링이 소프트웨어와 정보 시스템 분야로 확장됨
1990년대  시스템 엔지니어링 협회(INCOSE)가 창립됨
```


**INCOSE**  
: 시스템 엔지니어링 분야의 발전을 위해 1990년 설립  
: 시스템 엔지니어링 방법론의 표준화하여 시스템 엔지니어링 핸드북을 발행함
