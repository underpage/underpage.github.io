# Algorithm Design
: 주어진 문제를 해결하기 위해 체계적인 절차를 개발하고 최적의 방법을 찾아내는 과정  
: 절차를 개발하고 성능을 평가해 효율성과 정확성을 고려해야함

## 1. 문제 정의 및 분석
: 해결해야 할 문제를 이해하고 명확하게 정의하는 단계  
: 기존에 유사한 문제가 있는지 조사하고 해결 방법을 참고 할 수 있음  
: 데이터의 형태와 범위를 구체적으로 파악하고 입력 데이터와 출력 데이터를 정의함  
: 문제 해결에 필요한 추가 자원이나 제약에 대한 파악이 필요함  



## 2. 알고리즘 설계
: 문제를 해결하기 위한 핵심 절차를 구상하는 단계  
: 문제의 특성을 분석하여 적절한 접근법, 전략, 기법을 선택해 적용함  
: 기존 전략과 기법이 적절하지 않다면 문제를 재정의하거나 새로운 방법론을 설계하여 해결함  
: 알고리즘의 전체적인 흐름도가 파악되어야 하며 이때 예외 상황도 고려되어야 함

**접근법** (Approach)  
: 문제를 해결하기 위한 전체적인 방향을 결정하는 개념  

종류 | 설명 | 구체화
---|---|---
상향식 접근법 | 작은 문제부터 점진적으로 해결하는 방법 | 동적 프로그래밍
하향식 접근법 | 큰 문제를 작은 문제로 분해하여 해결하는 방법 | 분할 정복, 동적 프로그래밍
최적화 접근법 | 주어진 조건에서 최적의 해를 찾는 방법 | 탐욕적 방법, 분기 한정법
근사 해법 | 최적의 해를 찾기 어려운 경우 근사적인 해를 찾는 방법 | 휴리스틱 접근법, 몬테카를로 방법
확률적 접근법 | 무작위성과 확률을 활용하여 문제를 해결하는 방법 | 랜덤화 알고리즘, 몬테카를로 방법
시뮬레이션 접근법 | 실제 시스템을 모델링해 문제를 해결하는 방법 | 메타 휴리스틱 접근법



## 3. 알고리즘 표현 및 기술
: 설계한 알고리즘을 다양한 방법으로 표현하고 기술함  

종류 | 설명
---|---
자연어 | 사람의 언어로 표현해 이해하기 쉬우나 일관성이나 명확성이 떨어짐
순서도 | 각 단계를 도식화해 명령의 흐름을 쉽게 파악하나 복잡한 구조는 표현하기 어려움
의사코드 | 프로그래밍 언어와 유사한 형태로 표현하나 의사 코드는 정해진 문법이 없어 일관성이 떨어짐


**순서도** (Flowchart)

도형 | 설명
---|---
동그라미 | 순서도의 시작이나 종료 지점을 나타냄
평행사변형 | 입력이나 출력을 나타냄
마름모 | 조건이나 판단문을 나타냄
사각형 | 처리할 작업이나 명령을 나타냄
화살표 | 제어 흐름을 나타냄


**의사코드** (Pseudocode)  
: 알고리즘을 자연어와 프로그래밍 언어 중간 형태로 표현  
: 알고리즘 동작 방식을 설명하는 용도로 실제 실행되지 않으며 프로그래밍 언어로 변환해야 함  



## 4. 정확성 검증 
: 알고리즘이 모든 가능한 입력에 대해 올바른 결과를 생성하는지 확인  
: 귀납적 증명, 수학적 증명, 테스트 케이스 등을 통해 알고리즘의 정확성을 입증하고 보강함  
: 논리적 오류, 경계 조건, 예외 상황 등을 고려하고 알고리즘의 종료 조건이 명확한지 확인함  

**귀납적 증명**  
: 알고리즘이 모든 가능한 입력에 대해 올바른 결과를 생성하는지 수학적 귀납법을 사용해 증명  

종류 | 설명
---|---
기본 | 알고리즘이 가장 작은 입력에 대해 올바르게 작동하는지 확인
귀납 | 알고리즘이 n에 대해 올바르게 작동한다면 n+1에 대해서도 올바르게 작동하는지 증명


**수학적 증명**  
: 알고리즘이 정확히 동작하는지 논리적이고 체계적인 방법으로 증명  

증명 | 설명
---|---
대수적 증명 | 알고리즘의 동작을 수학적 등식과 부등식을 사용해 증명
모순 증명 | 알고리즘이 틀렸다고 가정하고 가정으로부터 모순을 도출해 올바름을 증명
구성적 증명 | 알고리즘이 올바른 결과를 도출하는 과정과 그 단계들을 구체적으로 증명


**테스트 케이스**  
: 알고리즘의 정확성을 다양한 입력에 대해 확인하는 방법  
: 의도적으로 실패 테스트를 하여 예외 상황 처리 여부를 검토할 수 있음  

방법 | 설명
---|---
단계별 검증 | 알고리즘의 각 단계가 올바르게 작동하는지 중간 결과를 확인하는 방법
테스트 자동화 | 다양한 입력에 대한 테스트를 자동화하여 광범위한 검증을 수행하는 방법
반례 찾기 | 알고리즘이 실패할 수 있는 특수 입력 케이스를 의도적으로 테스트하는 방법
불변 조건 | 알고리즘 실행 중 항상 참이어야 하는 조건을 정의하고 이 조건이 유지되는지 검증



## 5. 효율성 분석
: 알고리즘의 성능을 분석하고 평가하는 단계  
: 다양한 [알고리즘 분석](./algorithm-analysis.md)을 통해 알고리즘의 효율성을 분석하고 평가함  
: 일반적으로 시간 복잡도와 공간 복잡도를 통해 효율성을 평가하고 빅오 표기법을 사용해 성능을 평가함  

분석 | 설명
---|---
시간 복잡도 분석 | 알고리즘이 실행 시간에 대한 수학적 분석
공간 복잡도 분석 | 알고리즘이 사용하는 메모리 공간에 대한 분석
최악/평균/최선 분석 | 다양한 입력 시나리오에 따른 알고리즘 성능 분석
병목 현상 식별 | 가장 자원을 많이 소모하는 부분을 식별하고 최적화 가능성을 탐색



## 6. 최적화
: 분석 결과를 바탕으로 불필요한 연산이나 중복 처리를 제거해 알고리즘을 최적화하는 단계  
: 더 효율적인 자료구조나 알고리즘으로 대체 할 수 있는지 검토하고 최적화 후에는 성능 향상을 측정함  

**알고리즘 최적화**
- 불필요한 연산이나 사용되지 않는 결과값 제거
- 조건문 경우 빈도가 높은 조건을 먼저 검사하도록 순서 조정
- 반복문 내부에 불변 조건이 있다면 외부로 이동
- 꼬리 재귀 경우 반복문으로 변환 검토

**자료구조 최적화**
- 문제에 적합한 자료구조인지 검토하고 더 효율적인 자료구조로 교체
- 동적 메모리 할당을 최소화할 수 있도록 검토
- 자주 사용되는 데이터는 캐싱될 수 있도록 조정
- 데이터가 자주 검색되는 경우 인덱스를 사용할 수 있으면 활용하거나 추가



## 7. 구현 및 테스트
: 설계된 알고리즘을 프로그래밍 언어로 구현하고 다양한 입력에 대해 테스트하는 단계  

종류 | 설명
---|---
단위 테스트 | 함수나 모듈의 동작이 올바르게 동작하는지 검증
통합 테스트 | 전체 알고리즘이 올바르게 동작하는지 검증
예외 테스트 | 예외적인 입력이나 비정상적인 상황에 대해 적절하게 대응하는지 검증
경계값 테스트 | 경계에 해당하는 값(최소값과 최대값)을 입력해 처리할 수 있는지 검증
무작위 테스트 | 랜덤 데이터를 생성해 다양한 케이스에 대해 처리할 수 있는지 검증
동일성 테스트 | 같은 입력에 대해 항상 동일한 결과를 생성하는지 검증
