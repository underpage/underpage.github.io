# Data Collection
: 사용자 입력, 웹 크롤링, 센서 등 다양한 원천으로부터 체계적으로 데이터를 획득하는 과정  
: 단순한 데이터 획득을 넘어 데이터 분석 및 활용을 위해 형식, 품질, 신뢰성, 비용 등을 고려해야 함  
: 수집된 데이터의 품질과 안정성은 분석 결과에 직결되므로 수집 단계에서 검증되어야 함

**수집 프로세스**
```
목표 및 요구 사항 정의 > 데이터 소스 식별 > 수집 전략 설계 > 데이터 수집 > 데이터 적재
```

고려사항
- 데이터를 저장하거나 분석하기 알맞은 형태(포맷)로 수집해야 함 
- 수집된 데이터를 어떻게 통합하고 어디에 저장할지 고려되어야 함
- 수집된 데이터의 민감도에 따라 보안이나 개인정보 보호 조치를 수립해야 함

**데이터 적재**  
: 수집된 데이터를 다른 저장소나 시스템으로 이동하고 로드하는 프로세스  
: 수집된 데이터는 처리가 필요하며 이를 위해 하나의 저장소로 이동하여 로드/저장되어야 함  
: 이때 설계에 따라 최종 데이터 저장지에 적재되거나 임시 저장지에 적재될 수 있음  

- [데이터 적재](./data-ingestion.md)


**주요 용어**

용어 | 설명
---|---
데이터 볼륨 | 수집할 데이터의 크기
데이터 속도 | 데이터가 생성되거나 수집되는 속도
데이터 소스 | 데이터 원천이 되는 시스템이나 위치
데이터 구조 | 정형, 반정형, 비정형 데이터
데이터 민감도 | 데이터가 기밀 정보를 포함하는지 정도
데이터 신뢰성 | 데이터 소스의 가용성과 일관성
데이터 일관성 | 동일한 데이터가 여러 소스나 시점에서 동일한 값인지 여부
