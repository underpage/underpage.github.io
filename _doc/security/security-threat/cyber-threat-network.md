# Network Threat
: 네트워크를 통해 보안을 침해하거나 네트워크의 정상적인 동작을 방해하는 위협 요소들  
: 공격자의 의도와 방식에 따라 수동적 공격과 능동적 공격으로 구분됨  

**수동적 공격**
- [스캐닝](#scanning)
- [스니핑](#sniffing)

**능동적 공격**
- [스푸핑](#spoofing)
- [서비스 거부](#denial-of-service)
- [세션 공격](#session-attack)
- [중간자 공격](#man-in-the-middle)
- [라우팅 공격](#routing-attack)

유형 | 공격 방식 | 목적
---|---|---
스캐닝 | 호스트, 포트, 서비스 정보 수집 | 취약점 발견, 네트워크 구조 파악
스니핑 | 트래픽을 가로채어 패킷 분석 | 인증 정보 탈취, 데이터 도청
스푸핑 | IP, MAC, DNS 등의 정보 위조 | 신원 위장, 트래픽 가로채기 및 우회
서비스 거부 | 자원 고갈을 통한 서비스 마비 | 시스템 마비, 서비스 중단
중간자 공격 | 트래픽을 가로채어 정보 조작 | 데이터 도청 및 변조, 세션 탈취 
라우팅 공격 | 라우팅을 조작하여 경로 위조 | 트래픽 제어 및 우회



## Scanning
: 네트워크 상의 호스트, 포트, 서비스 등을 탐지해 네트워크 정보를 수집하는 행위  
: 네트워크를 공격하기 전에 공격 대상을 식별하거나 네트워크 취약점을 찾기 위해 사용됨  

**기법**
- 탐지 기법
  - 포트 스캐닝
  - 네트워크 스캐닝
  - 핑 스윕
- 분석 기법
  - OS 핑거프린팅
  - 서비스 버전 탐지
  - 취약점 스캐닝
- 우회 기법
  - 스텔스 스캔
  - 분산 스캔
  - 타이밍 조작
  - 디코이 스캔
  - 프래그먼트 스캔

**취약점**
- 불필요한 포트 개방
- 방화벽 정책 미흡
- 서비스 배너 정보 노출
- 시스템 정보 과다 공개
- 스캔 탐지 시스템 부재
- 로그 모니터링 미흡

**서비스 배너**  
: 네트워크 서비스가 연결 시 제공하는 정보로 서비스 버전, 운영체제 정보 등을 포함하는 응답 메시지
