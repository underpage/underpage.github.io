## Rule-based System
: 기호와 규칙 기반 지식 표현 및 추론 중심 접근법  
: 학습이 없으며 사전에 정의된 규칙과 추론 알고리즘을 통해 문제를 해결함  
: 해당 도메인 전문가의 지식이 규칙으로 직접 인코딩됨  
: 규칙이 없는 상황에 대해 추론이 불가능함 (유연성 부족)  

개념 | 설명
---|---
기호 | 현실 세계의 개념, 객체를 추상적으로 표현한 단위
규칙 | 조건과 결론 관계를 IF-THEN 형태로 명시하는 방식
지식 표현 | 인간의 지식을 컴퓨터가 처리할 수 있는 형태로 구조화
추론 | 기존 지식(규칙)으로부터 새로운 결론을 도출하는 과정


**구성요소**
- 지식 표현 체계 (프레임, 의미망, 온톨로지)
- 지식 베이스: 규칙을 저장하는 데이터베이스
- 추론 엔진: 규칙을 적용하여 결론을 도출하는 처리기


**알고리즘**
- 순방향 연쇄: 사실로부터 결론 도출 (데이터 주도 방식)
- 역방향 연쇄: 목표로부터 역추적하여 결론을 증명 (목표 주도 방식)
- 규칙 매칭: 사실과 규칙의 조건이 맞는지 확인
- 충돌 해결: 여러 규칙 적용 시 우선순위를 결정


**한계점**
- 문제 영역이 커지면 그만큼 규칙의 수가 늘어나 관리가 어려움
- 전문가의 지식을 규칙으로 인코딩하는 과정이 어렵고 시간이 오래 걸림
- 새로운 지식을 추가하면 기존 규칙들과 충돌하거나 시스템 동작에 영향을 미침
- 규칙 시스템은 예/아니오로 명확한 조건이 필요하지만 현실 문제는 그러하지 않음

=> 이러한 한계점으로 인해 규칙이 아니라 지식을 구조적으로 표현해 추론을 수행하는 연구가 이루어짐
