# Software Architecture Principle
: 소프트웨어 아키텍처 원칙이란 시스템의 전체적인 구조와 핵심 요소들을 설계할 때 따르는 고수준 지침  
: 시스템의 확장성, 재사용성, 유지보수성 등 품질 속성을 아키텍처 수준에서 지원하는 체계적인 접근 방식을 제공함  
: 설계 패턴이나 구현 기술을 넘어서 시스템의 장기적인 변화에 대응하는 구조적 토대를 확립함  

**원칙**
- 관심사 분리
- 제어의 역전
- 관례보다 구성

**특성**
- 무결성 : 아키텍처 전반에 일관된 규칙과 설계를 유지해야 함
- 투명성 : 시스템의 동작 방식과 구조가 명확히 드러나야 함
- 단순성 : 불필요한 복잡성을 피하고 가능한 한 간단한 해결책을 선택해야 함
- 가역성 : 중요한 아키텍처 결정을 쉽게 변경할 수 있어야 함
- 적응성 : 변화하는 요구사항과 환경에 유연하게 대응할 수 있어야 함
