# Type System
: 데이터에 타입을 부여해 데이터를 분류하고 관리하는 규칙 집합  
: 사용자에게 데이터에 대한 추상적 정보를 제공하고 프로그램이 데이터를 올바르게 사용할 수 있도록 함  
: 프로그램은 데이터 타입에 맞는 연산을 수행하므로 데이터의 정확성, 안전성, 효율성을 보장함  

- 타입 이론
- 타입 시스템 분류
- 타입 시스템 구성 요소
- 데이터 타입
- 타입 시스템 확장
- 최신 타입 시스템


**타입**  
: 타입이란 어떤 값을 다른 값과 분별할 수 있는 고유한 특성  

**컴퓨터에서 타입**  
: 메모리 공간에 할당된 데이터의 해석 방법과 조작 방법을 정의하는 속성  
: 타입은 데이터의 크기, 표현 방식, 연산을 결정하며 타입 시스템은 이러한 정보를 통해 데이터 접근과 조작을 제어함  
: 타입에 따른 메모리 접근 제한, 검증 등을 통해 메모리 안전성 문제를 예방함  

**history**
```
1900년대  버트런드 러셀이 집합 이론의 모순을 해결하기 위해 타입 이론 제시
1940년대  알론조 처치의 람다 계산법이 타입 시스템의 이론적 기초를 마련함

1950년대  포트란이 수치 계산의 효율성을 높이기 위해 부분적으로 타입 시스템 도입
1970년대  파스칼, C 등 언어에서 체계적인 타입 시스템이 도입됨
1980년대  하스겔, ML 등 언어에서 다형성과 타입 추론이 발전함
1990년대  객체지향 언어가 발전하며 클래스 기반 타입 시스템이 확산됨
2000년대  정적 타입과 동적 타입의 장점을 결합한 점진적 타입 시스템 등장
```
