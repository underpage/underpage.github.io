# Data Processing
: 데이터 처리란 원시 데이터를 의미 있고 유용한 정보로 변환하는 체계적인 과정  
: 데이터 수집, 변환, 저장, 분석 등 단계를 거쳐 활용 가능한 형태로 만드는 전체 데이터 흐름  

**processing vs preprocessing**  
: 데이터 처리는 데이터를 변형, 조작, 분석하는 포괄적인 과정  
: 데이터 전처리는 원시 데이터를 정제, 정리하는 초기 준비 단계  

**주요 단계**  
: 단계의 순서는 중요하지 않으며 설계나 필요에 의해 달라질 수 있음  

```
데이터 수집/변환 > 데이터 전처리/통합 > 데이터 가공/저장 > 데이터 분석 > 데이터 활용
```

## Data Processing Glossary
: 데이터 처리 과정에서 사용되는 개념 및 용어들  

**기본 개념**

용어 | 설명
---|---
데이터 셋 | 특정 목적을 위해 수집된 데이터 집합
원시 데이터 | 수집된 원본 데이터로 가공이나 정제가 이루어지지 않은 상태
출발지(Source) | 데이터가 생성되거나 저장된 원본 시스템
도착지(Target) | 처리된 데이터가 저장되거나 활용될 시스템


**데이터 속성**

용어 | 설명
---|---
속성 | 데이터 타입, 분포, 범위 등 데이터가 가지는 기본적인 특성
구조 | 데이터 정렬 방식, 계층 구조, 차원성 등 데이터의 구성 방식
관계 | 데이터 간의 상관 관계, 의존성 등 상호작용과 연결성


**데이터 구조**

용어 | 설명
---|---
정형 데이터  | 행과 열이 명확한 표 형태로 고정된 구조가 있는 데이터
반정형 데이터 | 일정한 구조가 있지만 유연한 데이터
비정형 데이터 | 미리 정의된 구조가 없는 데이터


**데이터 유형**

용어 | 설명
---|---
연속형 | 임의의 구간에서 무한히 많은 값을 가질 수 있는 데이터
범주형 | 제한된 수의 구분된 범주나 그룹으로 분류될 수 있는 데이터
이산형 | 셀 수 있는 별개의 값을 가지는 데이터
시계열 | 시간 순서에 따라 기록된 데이터


**데이터 특성**  
: 데이터 특성은 데이터 처리 단계마다 다르게 정의될 수 있으며 아키텍처나 기법 결정에 영향을 미침  

용어 | 종류
---|---
데이터 볼륨 | 데이터의 양 (소규모, 대용량)
데이터 적시성 |	데이터 수집과 처리 시점 (실시간, 비실시간)
데이터 출처 | 데이터의 원천 (내부 데이터, 외부 데이터)
데이터 변동성 | 데이터 변화 정도 (정적, 동적)
데이터 공간성 | 데이터의 지리적, 공간적 특성
데이터 희소성 | 데이터에 0 또는 결측값이 많은 정도
데이터 차원 | 데이터가 가진 변수나 특성의 수 (고차원, 저차원)
데이터 복잡성 | 데이터 패턴의 복잡도 (선형성, 비선형성)
데이터 시간성 |	데이터의 시간적 특성 (정적, 동적, 이력 정보)


**데이터 품질**

용어 | 설명
---|---
중복 데이터 | 동일한 값이 여러 번 나타나는 상태
불일치 데이터 | 데이터 내 모순되거나 일관성이 없는 값
결측치 | 데이터 셋에 어떤 값이 누락된 상태
이상치 | 일반적인 패턴에서 크게 벗어난 값
노이즈 | 데이터에 포함된 오차나 변동


**데이터 기법**

용어 | 설명
---|---
프로파일링 | 데이터의 패턴, 품질, 특성 등을 분석해 특성과 문제점을 파악하는 기법
