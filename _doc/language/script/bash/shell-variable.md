# Shell Variable
: 관례상 환경 변수는 대문자로 사용자 정의 변수는 소문자를 사용함  

- 환경 변수
- 사용자 정의 변수
  - 외부 변수
  - 전역 변수
  - 지역 변수
  - 파라미터 변수


변수 종류 | 스코프 | 설명
---|---|---
환경 변수 | 전역 스코프 | 현재 쉘과 모든 하위 프로세스에서 접근 가능
전역 변수 | 쉘 스코프 | 현재 쉘 세션으로 범위가 제한됨 (하위 프로세스 접근 불가)
지역 변수 | 함수 스코프 | 정의된 함수 내에서만 접근 가능



## Environment Variable
= 예약 변수  
: 시스템에서 기본으로 제공하는 변수로 모든 프로세스에서 접근 가능함  
: export 명령어를 통해 사용자 정의 환경 변수를 정의할 수 있음    

```bash
# 환경 변수 확인
echo $환경변수

# 환경 변수 값 변경 또는 새 환경 변수 생성
export 환경변수=값

# 모든 환경 변수 확인
env
```


**주요 환경 변수**

환경 변수 | 설명
---|---
HOME  | 현재 사용자의 홈 디렉토리
USER  | 현재 로그인한 사용자 이름
SHELL | 현재 사용 중인 쉘 경로



## User-defined Variable
: 사용자가 직접 정의하여 쉘 스크립트에서 활용하는 변수  
: 변수 사용 전 미리 선언하지 않으면 처음 값을 할당해 자동으로 변수가 생성됨  
: 기본적으로 모든 변수의 값은 문자열로 취급됨  

```bash
# 변수 선언
변수=값

# 변수 출력
echo $변수

# 변수 삭제
unset $변수

# 모든 변수 목록 출력
set
```

변수 할당 | 설명
---|---
${변수:-기본값} | 변수가 설정되지 않았거나 빈 문자열일 때 기본값 사용
${변수:=기본값} | 변수가 설정되지 않았거나 빈 문자열일 때 기본값을 할당하고 사용
${변수:+대체값} | 변수가 설정되었을 때만 대체값 사용
${변수:?메시지} | 변수가 설정되지 않았거나 빈 문자열일 때 메시지 출력 후 종료


**declare**  
: declare 또는 typeset 명령어를 사용해 변수의 속성을 설정할 수 있음  

옵션 | 설명
---|---
-i | 변수를 정수로 선언 
-r | 변수를 읽기 전용으로 만듬 (readonly)
-x | 변수를 환경으로 내보냄 (export)
-a | 변수를 배열로 선언
-A | 변수를 연관 배열로 선언

```bash
# 변수를 정수로 선언
declare -i num=0

# 변수를 배열로 선언
declare -a arr 
arr[0]=1
arr[2]=2

echo ${arr[0]}
```



### Exported Variable
: export 명령어를 사용해 변수를 환경 변수로 만들 수 있음  
: 부모 프로세스에서 자식 프로세스로 변수가 전달됨  

```sh
#!/bin/sh
export var="외부 변수"
```



### Global Variable
: 현재 세션에서만 유지되는 변수  
: 함수 외부에서 선언된 변수는 기본적으로 전역 변수  

```sh
#!/bin/sh
var="전역 변수"
```



### Local Variable
: 함수 내부에서 local 명령어로 정의된 함수  
: 함수 내부에서만 유효하며 함수 실행이 종료되면 소멸됨  

```sh
#!/bin/sh
myFunction() {
  var="전역 변수"
  local local_var="지역 변수"
}
```



### Parameter Variable
: 스크립트나 함수에 전달된 인자값을 저장하는 변수  

변수 | 설명
---|---
$0  | 실행된 스크립트 이름 
$1 ~ $9, ${10} | 위치에 따른 인자값
\$* | 모든 인자를 하나의 문자열로 처리
\$@ | 모든 인자를 개별 문자열로 처리
\$# | 총 인자 개수 


**shift**  
: shift 명령어를 사용해 파라미터 변수를 왼쪽으로 한 단계씩 시프트함  
: $1은 사라지고, $2는 $1이 되고, $3은 $2가 되고, $#는 1 감소함
