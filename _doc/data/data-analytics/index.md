# Data Analytics
: 수집 또는 저장된 데이터를 통해 문제를 해결하거나 인사이트를 도출하는 기술 및 체계  
: 데이터로부터 유용한 정보를 추출하고 결론을 도출하여 의사 결정을 지원하는 과정  
: 데이터 분석을 위해서는 도메인 이해와 데이터를 다루는 통계적 지식 및 도구 활용 능력이 필요함  

**분석 발전 과정**  
: 데이터 분석은 다양한 기술과 데이터 환경의 발전에 따라 진화함  

```
규칙 기반 분석 > 통계 기반 분석 > 데이터 마이닝 > 기계학습 > 딥러닝 > 자동화된 기계학습
```

- 전통적 통계 분석: 확률론과 가설 검정 기법을 통해 데이터를 설명적으로 해석함
- 도구 기반 분석: DBMS, BI 도구를 활용해 구조화된 데이터를 분석하고 대시보드 중심의 의사결정이 가능해짐
- 머신러닝 기반 분석: 데이터 규모와 다양성 증가로 패턴 학습 및 예측적 분석이 가능해짐
- 실시간/지능형 분석: 실시간 스트리밍 분석, 강화학습 기반 의사결정, 인과추론 기반 분석


**분석 유형**  
: 데이터 분석의 목적에 따라 분류됨  

구분 | 설명
---|---
설명적 분석 | 과거 데이터를 요약해 무엇이 일어났는지 파악함 (평균, 빈도수)
진단적 분석 | 왜 일어났는지를 탐색하고 인과 관계를 분석함
예측적 분석 | 과거 패턴을 통해 앞으로 무슨 일이 일어날지 예측함
처방적 분석 | 앞으로 무엇을 해야하는지 제시함


**분석 프로세스**
```
문제 정의 및 목표 설정 > 데이터 이해 > 분석 기법 선택 > 분석 수행 > 결과 해석 및 인사이트 도출
```

1. 해결하고자 하는 문제를 정의하고 분석 목표를 수립함
2. 데이터의 구조, 품질, 특성, 한계 등을 파악함
3. 분석 목표(설명/진단/예측/처방)에 적합한 분석 기법을 선택함
4. 통계 분석, 모델링, 시각화 등을 통해 구체적인 분석을 수행함
5. 결과를 해석하고 인사이트를 도출하여 의사결정에 반영함


**데이터 이해**  
: 데이터를 파악하여 고려사항을 판별하고 분석의 방향성을 결정함  
: 데이터 품질은 분석의 정확성을 좌우하는 핵심 요소로 데이터 특성을 이해하는 과정이 중요함  

관점 | 설명
---|---
거시적 상태 | 데이터셋 전체가 어떤 상태인지 (크기, 구조, 결측률, 중복률 등)
미시적 상태 | 변수(컬럼)가 어떤 값을 가지는지 (값 분포, 이상치 등)
컨텍스트 | 데이터가 왜 이런 특성을 형성하는지 (수집 방식, 처리 이력, 도메인 등)
