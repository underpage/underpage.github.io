```
알고리즘은 1960년대 컴퓨터 과학이 발전하며 함께 발전

```


**구현 기법** (Implementation Techniques)  
: 알고리즘 설계 기법을 실제 코드로 변환하는 구체적인 방법이나 패턴  

종류 | 설명 | 사용
---|---|---
반복 | 명시적 반복문을 통한 문제 해결 | 동적 프로그래밍, 탐욕적 방법
재귀 | 문제를 더 작은 부분 문제로 분할하여 해결 | 분할 정복, 동적 프로그래밍
캐싱 | 중복 계산 방지를 위해 결과를 저장해 재사용함 | 동적 프로그래밍
분할 | 문제를 더 작은 부분 문제로 나누어 해결 | 분할 정복
탐색 | 가능한 해를 체계적으로 탐색해 해결 | 백트래킹
탐욕적 선택 | 각 단계에서 최적의 해를 선택하여 문제 해결 | 탐욕적 방법
휴리스틱 | 최적의 해결책을 빠르게 찾기 위해 경험적 방법을 사용 | 휴리스틱 접근법
무작위 샘플링 | 난수를 활용하여 해를 선택하거나 근사 해를 찾음 | 몬테카를로 방법
확률적 탐색 | 확률적 요소를 포함하여 탐색을 수행 | 몬테카를로 방법



## Divide and Conquer
분할 정복 방법 = 하향식 접근법  
: 문제를 작은 문제로 나눈 후 작은 문제들을 각각 해결하고 그 결과를 결합해 원래 문제를 해결함  
: 분할된 문제는 하나의 독립성을 가지며 문제의 성격과 특징이 원래 문제와 동일해야함  

**단계**
```
분할된 문제들은 다음 과정을 거침

분할 : 문제를 작은 문제로 나눔
정복 : 작은 문제를 해결함
결합 : 작은 문제들의 결과를 결합함
```

**알고리즘**
- 이진 탐색
- 병합 정렬
- 퀵 정렬
- 스트라센 행렬 곱셈



## Dynamic Programming
동적 프로그래밍 = 상향식 접근법  
: 문제를 작은 문제로 나누어 해결하고 그 결과를 저장해 중복된 계산을 피하는 방법  
: 작은 문제들을 해결하면서 결과를 저장하고 그 결과를 활용해 최적의 해를 구하는 상향식 접근법  
: 작은 문제들은 독립적일 필요가 없으며 중복 계산을 방지하기 위해 메모이제이션 기법을 사용함  

**메모이제이션**  
: 중복 계산을 방지하기 위해 계산된 결과를 메모리에 저장하고 그 결과를 재사용하는 방법  

**알고리즘**
- 플로이드-워셜 알고리즘
- 벨만-포드 알고리즘
- 최장 증가 부분 수열
- 최장 공통 부분 수열



## Branch and Bound
분기 한정법  
: 탐색 공간을 분할하고 한계를 설정하여 불필요한 탐색을 제거하는 방법  
: 백트래킹과 유사하지만 각 단계에 한계를 정해 불필요한 탐색을 줄임  

**알고리즘**
- 분기 한정 알고리즘



## Brute Force
브루트 포스, 무작위 대입  
: 가능한 모든 해결책을 시도해 문제를 해결하는 방법  

**알고리즘**
- KMP 알고리즘
- 완전 탐색 기반 소수 판별



## Backtracking
: 문제를 풀다가 막히면 되돌아가서 다시 문제를 푸는 방법  
: 문제를 풀기 위해 모든 가능성을 시도하다가 막히면 되돌아가서 다른 방법을 찾음  

**알고리즘**
- N-Queens 문제 해결 알고리즘
- 해밀턴 경로 탐색
- 부분 집합 합 문제



## Greedy Technique
탐욕적 방법, 욕심쟁이 방법  
: 각 단계에서 가장 최적의 선택을 하여 전체 문제를 해결하는 방법  
: 각 단계에서는 최적의 해일 수 있으나 전체 문제를 해결하는 최적의 해라는 보장은 없음  

**알고리즘**
- 크루스칼 알고리즘
- 프림 알고리즘
- 다익스트라 알고리즘
- 허프만 코딩



## Heuristic Approach
휴리스틱 접근법
: 문제 해결을 위해 근사적인 해결을 찾는 방법  
: 최적해를 보장하지 않지만 빠른 시간 내에 해결책을 찾는 방법  

**알고리즘**
- A* 알고리즘
- 유전 알고리즘
- 시뮬레이티드 어닐링



## Monte Carlo Technique
몬테카를로 방법  
: 난수를 활용하여 확률적으로 해를 구하는 방법  

**알고리즘**
- 랜덤 트리 탐색
- 몬테카를로 적분
