# Data Processing Architecture
: 데이터 처리 시스템의 전체적인 구조와 접근 방식

**중앙집중식 아키텍처**  
: 모든 데이터를 하나의 중앙 저장소에 저장하고 관리하는 구조  
: 일반적으로 정형 데이터를 ETL(Extract, Transform, Load) 방식으로 처리함  
: 데이터 일관성이 보장되나 확장성과 실시간 처리가 어려우며 비정형 데이터 저장이 제한됨  
: 확장성이란 시스템이 증가하는 데이터 볼륨, 사용자 수 등에 효율적으로 대응할 수 있는 능력을 의미함  

- 메인프레임 데이터
- 단일 데이터베이스
- 데이터 웨어하우스 아키텍처


**분산 데이터 아키텍처**  
: 중앙집중식 아키텍처의 한계를 극복하기 위해 데이터를 여러 저장소에 분산하는 방식  
: 확장성 및 유연성이 뛰어나나 데이터 일관성과 품질 관리가 어려울 수 있음  
: 일관성이란 분산된 모든 저장소에서 동일한 데이터를 유지하는 특성을 의미함  

일관성 모델 | 설명
---|---
강한 일관성 | 모든 노드가 즉시 동일한 데이터를 보장하며 이는 성능 저하 가능성을 내포함
약한 일관성 | 노드 간 데이터 동기화를 즉시 수행하지 않아 일관성을 보장하지 않으나 성능도 저하되지 않음
최종적 일관성 | 시간이 지나면 모든 노드가 동일한 데이터를 가지도록 보장하며 일시적으로 데이터가 불일치 상태


- 데이터 레이크 아키텍처
- 람다 아키텍처
- 카파 아키텍처
- 데이터 패브릭 아키텍처
- 데이터 메시 아키텍처

```
현대 데이터 아키텍처는 물리적 분산과 논리적 중앙화라는 이중적 특성을 가짐
```


**하이브리드 아키텍처**  
: 중앙집중식과 분산형을 혼합한 형태로 여러 아키텍처를 조합해 사용하는 방식  
: 여러 기술이 조합되어 운영 복잡성이 증가하며 데이터 일관성, 보안 등 관리가 어려움
