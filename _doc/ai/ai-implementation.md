# AI Implementation
: 인공지능을 구현하기 위한 주요 패러다임과 구조적 접근법  

**발전 순서**
```
규칙 기반 > 초기 신경망 > 지식 기반 > 통계 기반 (전통적 머신러닝) > 현대 신경망
```

- 규칙 기반 시스템과 초기 신경망은 비슷한 시기에 발전함
- 머신 러닝은 통계 기반으로 시작해 신경망을 포함하여 현재는 두 갈래로 존재함
- 현재 인공지능 시스템은 여러 방법론과 기술을 혼합하여 뚜렷하게 구분되지 않음


**분류**
- [규칙 기반 시스템](#rule-based-system)
- [지식 기반 시스템](#knowledge-based-system)
- [통계 기반 시스템](#statistical-based-system)
- [신경망 기반 시스템](#neural-network-based-system)

영역 | 특징
---|---
규칙 기반 | 규칙을 기반으로 논리적 추론 수행
지식 기반 | 구조화된 지식으로 표현해 복잡한 추론 수행
통계 기반 | 데이터를 기반으로 패턴과 확률을 학습해 확률적 판단을 수행
신경망 기반 | 데이터를 학습해 패턴을 기반으로 예측 수행


기술 특성에 따라 다음과 같이 분류하기도 함
- 기호 기반 (규칙, 지식)
- 확률 기반 (통계, 확률)
- 연결 기반 (신경망)



## Rule-based System
: 사전에 정의된 규칙과 추론 알고리즘을 통해 문제를 해결  
: 해당 도메인 전문가의 지식을 규칙으로 직접 인코딩함  
: 규칙이 정의되지 않은 상황에서는 새로운 추론이 불가능함 (유연성 부족)  

용어 | 설명
---|---
기호 | 현실 세계의 개념, 객체를 추상적으로 표현한 단위
규칙 | 조건과 결론 관계를 IF-THEN 형태로 표현한 방식
지식 표현 | 인간의 지식을 컴퓨터가 처리할 수 있는 구조로 표현
추론 | 기존 지식(규칙)으로부터 새로운 결론을 도출하는 과정


**구성요소**
- 지식 베이스: 규칙을 저장하는 저장소
- 추론 엔진: 규칙을 적용하여 결론을 도출하는 처리기


**알고리즘**
- 순방향 연쇄: 사실로부터 결론 도출 (데이터 주도 방식)
- 역방향 연쇄: 목표로부터 역추적하여 결론을 증명 (목표 주도 방식)
- 규칙 매칭: 사실과 규칙의 조건이 맞는지 확인
- 충돌 해결: 여러 규칙 적용 시 우선순위를 결정


**한계점**
- 문제 영역이 커지면 그만큼 규칙의 수가 늘어나 관리가 어려움
- 전문가의 지식을 규칙으로 인코딩하는 과정이 어렵고 시간이 오래 걸림
- 새로운 지식을 추가하면 기존 규칙들과 충돌하거나 시스템 동작에 영향을 미침
- 규칙 시스템은 예/아니오로 명확한 조건이 필요하지만 현실 문제는 그렇지 않음

=> 이러한 한계로 인해 규칙을 넘어 더 복잡하고 구조화된 지식을 표현해 추론을 수행하는 연구가 이루어짐



## Knowledge-based System
: 특정 도메인 지식을 체계적으로 모델링하고 형식적 추론을 통해 문제를 해결  
: 단순한 규칙을 넘어 개념 간 관계와 계층 구조를 명시적으로 표현함  
: 구조적 접근법을 사용해 지식의 재사용성과 확장성을 높임  
: 명시적 지식을 바탕으로 추론을 통해 암시적 지식 도출이 가능함  

용어 | 설명
---|---
지식 | 문제 해결에 필요한 사실, 규칙, 개념 간 관계 등을 구조적으로 표현한 정보
명시적 지식 | 시스템에 입력되는 명확하게 표현된 지식
암시적 지식 | 추론을 통해 도출되는 새로운 지식
논리적 추론 | 형식 논리 규칙을 사용해 기존 지식에서 새로운 지식을 도출하는 과정
형식적 추론 | 수학적으로 정의된 추론 규칙을 사용하는 체계적인 추론 방법


**구성요소**
- 지식 표현 체계 (프레임, 의미망, 온톨로지)
- 지식 베이스 : 도메인 지식과 메타 지식을 저장하는 저장소
- 지식 획득 모듈
- 추론 엔진

용어 | 개념
---|---
프레임 | 상황이나 객체를 속성과 값으로 구조화하여 표현하는 방식
의미망 | 개체와 관계를 그래프 구조로 표현
온톨로지 | 의미망보다 더 체계적이고 형식적으로 지식을 정의하는 방식

=> 지식 표현 기술은 인공지능만 아니라 웹, 검색 엔진 등의 핵심 기술로 사용됨


**하위 범주**
- 추론 기반 시스템
- 탐색 기반 시스템


**문제점**
- 전문가 지식을 시스템화하는 과정이 복잡하고 비용이 높음
- 모호하고 불확실한 지식은 정확하게 표현하기 어려움
- 인간의 직관적인 상식을 명시적으로 표현하기 어려움
- 동일한 지식이라도 상황에 따라 다르게 해석될 수 있음

=> 이러한 한계로 인해 지식 표현 대신 데이터로부터 패턴을 학습하는 통계적 접근법이 주목받게 됨



## Statistical-based System
: 확률과 통계론을 기반으로 데이터로부터 규칙(패턴)을 학습하여 예측과 의사결정을 수행함  
: 데이터를 수치적 특성으로 변환하여 확률 모델로 표현하고 이를 분석해 일반화 가능한 규칙을 도출함

```
통계학은 17-18세기 확률론에서 시작되어 19-20세기에 체계적으로 발전한 수학 분야로 
20세기 중반 컴퓨터 시스템에 통계적 방법론을 도입하여 인공지능 연구의 이론적 기반을 제공함
```


**동작 과정**
```
데이터 수집 > 특성 추출 > 모델 선택/설계 > 학습 > 검증 > 예측
```

1. 해결하고자 하는 문제와 관련된 데이터를 수집함
2. 데이터를 분석 가능한 형태로 수치화하고 특성을 추출함
3. 문제 유형에 맞게 확률 모델 구조를 설계하거나 기존 모델에서 선택함
4. 훈련 데이터를 사용해 모델이 데이터 분포와 패턴을 찾도록 학습함 (추청된 매개변수 도출)
5. 검증 데이터를 사용해 학습된 모델의 예측 성능과 일반화 성능을 확인함
6. 학습된 모델로 새로운 상황에 대한 결과를 예측해


**주요 기술**
- 확률 모델
  - 선형 모델, 로지스틱 모델, 마르코프 모델, 베이지안 네트워크 모델
- 학습 기법
  - 회귀 분석, 분류 기법, 군집화 기법
- 추론 방법
  - 베이지안 추론, 최대우도추정, 최대사후확률추정
- 평가 방법
  - 교차 검증, 부트스트래핑

=> 이러한 기술들은 머신러닝과 딥러닝에 이론적 토대를 제공함


**한계점**
- 인간의 직관적 상식이나 맥락적 사실은 수치화하기 어려움
- 학습을 위해 대량의 데이터와 이를 처리하기 위한 연산 자원이 필요함
- 데이터 차원이 높아질수록 확률 분포를 정확하게 추정하기 어려움 (차원의 저주)
- 모델이 훈련 데이터에만 특화되어 새로운 데이터에 대한 예측은 어려움 (과적합)
- 데이터가 수치화되면서 정보가 손실되고 예측은 가능해도 결과에 대한 설명은 어려움
- 복잡한 관계를 표현하려면 모델이 복잡해지고 성능이 제한됨



## Neural Network-based System
: 인공 신경망 구조와 학습 알고리즘을 통해 데이터 패턴을 학습하여 문제를 해결  
: 뉴런(노드) 간 가중치 연결과 역전파 학습을 통해 데이터 입출력 매핑을 자동으로 학습함  
: 이미지, 텍스트, 음성 등 다양한 형태의 데이터에서 사람이 찾기 어려운 복잡한 패턴을 발견함  


**범주**
- 초기 신경망
- 현대 신경망


**학습 과정**
```
순전파 > 손실 계산 > 역전파 > 가중치 업데이트 > 반복 학습
```

1. 입력층 > 은닉층 > 출력층 순으로 데이터를 처리함
2. 예측값과 정답 간 차이를 손실 함수로 측정함
3. 오차를 출력층 > 은닉층 > 입력층 순으로 전파해 가중치의 기여도를 계산함
4. 계산괸 정보를 바탕으로 가중치를 업데이트함
5. 설정한 횟수(에폭) 만큼 위 과정을 반복함


**구성요소**
- 신경망 아키텍처
- 활성화 함수
- 손실 함수
- 최적화 알고리즘



## Hybrid-based System
: 규칙 기반, 통계 기반, 신경망 기반 등 다양한 방법론을 활용해 문제를 해결  
: 각 접근 방식의 장점을 결합하여 성능과 유연성을 극대화함  

**구성요소**
- 통합 제어: 다양한 서브 시스템을 관리함
- 지식 융합: 서로 다른 지식 표현을 통합함
- 상황 인식
- 의사결정
